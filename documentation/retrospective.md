# Retrospective 

## Dobbeltdokumentation
I projektets spæde start virkede GitHub som en lidt uoverskuelig platform at skulle gå i gang med at lære at bruge. Vi noterede derfor alle vores tanker og arbejde i Google Docs. Da vi kom længere i processen og fik os sat ind i GitHub, begyndte vi at forstå platformen og dens funktioner, hvilket vi endte med at synes var ret fedt. Vi kopierede derfor vores tekst fra Docs-dokumenterne over i markdown-filer i GitHub og begyndte at samle alle elementerne i projektet herinde. Alt i alt har GitHub vist sig at være et fint program, som vi kan se god værdi i. 

## Roadmap/Scrum i GitHub Projects
Vores indledende tilgang til projektet var som sagt at samle alt angående projektet i Docs – herunder vores to-do og plan for projektet. Vi gav os senere i kast med at forsøge at lave og bruge Roadmaps og Scrum-boards i GitHub Projects, men vi fandt ikke dette ligeså intuitivt som at have det i Docs – vi endte derfor lidt med at lave en 'workaround', hvor vi er endt med at have samlet nogle af vores to-do punkter i GitHub og nogle andre i Docs, hvilket bestemt ikke har været en optimal måde at gøre det på. Skulle man lave et lignende projekt igen, ville det klart være en fordel kun at samle Roadmap/Scrum-board ét sted og kun køre det der. 


## Roller
Alle tre af gruppens medlemmer er nybegyndere rent programmeringsmæssigt, og HTML er dermed også noget helt nyt og ukendt for alle. Det tog derfor lidt tid at etablere konkrete roller og ansvarsområder, og i starten arbejdede vi meget sammen alle tre og udførte tingene via. én computer. Løbende i processen begyndte vi at forsøge os med hver vores arbejdsområder, og da vi blev mere og mere fortrolige med GitHub og logikken bag, begyndte vi at kunne arbejde mere selvstændigt og tildele hinanden egne ansvarsområder, som vi kunne lave hver for sig. 

## Opgaven
Generelt synes vi at projektet har været spændende, og da vi rent faktisk fik regnet GitHub og HTML osv. ud, så endte vi med at synes, at det var et sjovt og hyggeligt projekt at gennemføre. Vi oplevede meget få til ingen gruppemæssige konflikter og har haft et godt samarbejde, hvor alle har bidraget og  gjort et godt stykke arbejde (hvis vi selv skal sige det😊).
